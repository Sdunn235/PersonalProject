# AI Memory Architecture

## Purpose

Memory is what transforms NPCs from reactive systems into evolving entities.

Without memory:
- No identity
- No learning
- No relationships

---

## Memory Types

### 1. Event Memory
Stores:
- What happened
- Where it happened
- Who was involved
- Outcome

Example:
"Attacked by wolf near river — dangerous area"

---

### 2. Location Memory
Stores:
- Known places
- Resource locations
- Safe zones
- Dangerous zones

---

### 3. Entity Memory
Stores:
- Other NPCs
- Players
- Creatures

Includes:
- Trust level
- Past interactions
- Role classification

---

## Memory Influence on Behavior

Memory modifies:
- Decision weighting
- Risk tolerance
- Social interaction

Example:
- If NPC remembers betrayal → avoids that entity
- If NPC remembers food source → prioritizes that path

---

## Relationship Modeling

Relationships are built from:
- Repeated interactions
- Memory accumulation
- Outcome weighting

No fixed "friend/enemy" flags.

Relationships evolve.

---

## Persistence

Memory must:
- Persist across sessions
- Influence future world states
- Be queryable efficiently

---

## Memory Decay

Not all memory persists equally.

Factors:
- Importance
- Recency
- Emotional weight (future system)

---

## Open Areas

- Emotional tagging system (not finalized)
- Memory compression strategies at scale
- Long-term generational memory inheritance

---

## Non-Negotiable Rule

If memory does not affect future decisions,
it is useless and should not exist.
